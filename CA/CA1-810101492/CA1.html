import numpy as np

def binomial(size, n, p):
    choices = np.random.choice([0, 1], size=size * n, replace=True, p=[1-p, p])
    bin_array = np.array(choices).reshape(size, n)
    row_sums = np.sum(bin_array, axis=1)
    return row_sums

display(binomial(size = 5000, n = 500 , p = 0.5))

When we look at the extremes (probability values of 0 and 1), the practical and theoretical variances are quite similar. However, when we consider the middle point (probability value of 0.5), the difference between these variances becomes more noticeable.

Difference Between Practical and Theoretical Averages¶

As you can see, the practical and theoretical averages are incredibly close to each other, almost indistinguishable! This can be attributed to the abundance of trials conducted (a whopping 5000 trials). If the number of trials were significantly smaller, let's say just 5 trials, the differences between the two would become more noticeable. Hence, we can draw the conclusion that as the number of trials approaches infinity, the gap between practical and theoretical calculations disappears entirely. This holds true for variance as well!

2_1:¶

In [ ]:
import numpy as np
import matplotlib.pyplot as plt
from scipy.stats import binom, poisson, norm
from math import sqrt

size = 5000
n = 250
p = 0.008

lambda_poisson = n * p

mean_normal = n * p
std_normal = sqrt(n * p * (1-p))

binomial_data = np.random.binomial(n, p, size)
poisson_data = np.random.poisson(lambda_poisson, size)
normal_data = np.random.normal(mean_normal, std_normal, size)

plt.figure(figsize=(10, 5))

unique_binomial, counts_binomial = np.unique(binomial_data, return_counts=True)
frequency_for_binomial = counts_binomial / size
plt.plot(unique_binomial, frequency_for_binomial,marker='o', linestyle='-', color='blue', label='Binomial')

unique_poisson, counts_poisson = np.unique(poisson_data, return_counts=True)
frequency_for_poiss = counts_poisson / size
plt.plot(unique_poisson, frequency_for_poiss, marker='o', linestyle='-', color='green', label='Poisson')

x_values_normal = np.linspace(0, max(normal_data), 100)
normal_pdf_values = norm.pdf(x_values_normal, mean_normal, std_normal)
plt.plot(x_values_normal, normal_pdf_values, linestyle='-', color='red', label='Normal')

plt.title('Comparing Binomial, Poisson, and Normal Distributions')
plt.xlabel('Value')
plt.ylabel('Probability Density / Normalized Frequency')

plt.legend()
plt.show()

2_2:¶

In this case, we need to choose between two estimation methods: the Poisson distribution and the normal distribution. The Poisson distribution is a better choice for estimating the binomial distribution in this situation.

Here's why: The probability of dying in each collision is very low, close to zero at 0.008. There are a high number of collisions, around 250. When we multiply these values (250 * 0.008), we get a constant value of 2. This is important because it's a good reason to use the Poisson distribution for estimation.

If we tried to use the normal distribution, we would expect the probability of dying to be around 0.5, which is much higher than the actual probability. Since the probability is closer to zero than to the middle point, the Poisson distribution is a better choice for estimating the binomial distribution.

In the graphs provided, you may have noticed that I set the value of "mu" to 10. This choice was based on the assumption that it would serve as a good average for scores ranging from 0 to 20. Additionally, I set the value of "sigma" to 3.3, which represents the standard deviation. This decision was made because the majority of numbers fall within the range of 10 - 3 * 3.3 to 10 + 3 * 3.3. In other words, it is reasonable to assume that around 99.7 percent of scores lie between 0.1 and 19.9.

By observing the diagrams, we can see that they resemble a normal distribution, although they appear to have different standard deviations. This observation supports the Central Limit Theorem (CLT). Specifically:

The uniform distribution encompasses values between 0 and 20.

The exponential distribution has the value of "mu."

The Poisson distribution also aligns with the value of "mu."

The normal distribution, in this case, has a "mu" of 10 and a standard deviation of 3.3.

4_2:¶

Based on the plots, it can be observed that the normal distribution is a more accurate estimation for the binomial distribution compared to the Poisson distribution in this case. This is because when the probability of getting wounded is around the midpoint (p = 0.45, close to p = 0.5), the normal distribution provides a better approximation for the binomial distribution. On the other hand, the Poisson distribution is more suitable for probabilities close to zero in the context of the binomial distribution. Therefore, in this particular scenario, the normal distribution is the preferable choice for estimating the binomial distribution.
